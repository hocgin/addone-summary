# Popup 模型初始化进度展示

## 功能概述

Popup UI 中现在可以实时展示模型初始化进度，包括：

1. **实时进度条** - 显示整体加载进度
2. **当前阶段** - 显示正在执行的操作
3. **详细下载信息** - 显示下载速度、剩余时间等
4. **步骤指示器** - 显示当前处于哪个大步骤

## 进度展示流程

### 1. 用户点击"开始分析"

```
Popup 发送 SUMMARIZE_PAGE 消息
    ↓
Background 接收并开始处理
    ↓
发送进度更新到 Popup
```

### 2. 模型初始化阶段（0-30%）

```
初始化 WebGPU 环境 (0-10%)
    ├── 检查 WebGPU 支持
    └── 创建 GPU 上下文

加载模型配置 (10-20%)
    └── 读取模型元数据

下载模型权重文件 (20-50%)
    ├── 显示已下载大小
    ├── 显示下载速度
    └── 计算剩余时间

编译模型到 WebGPU (50-80%)
    └── 着色器编译

准备推理引擎 (80-100%)
    └── 完成初始化
```

### 3. 内容提取阶段（30%）

```
注入 Content Script
    ↓
提取页面文本内容
    ↓
清理和格式化
```

### 4. 摘要生成阶段（50-100%）

```
发送文本到 WebLLM
    ↓
流式接收响应
    ↓
解析 JSON 结果
```

## UI 组件说明

### LoadingView 组件

显示以下信息：

```tsx
<LoadingView
  progress={75}              // 进度百分比 (0-100)
  message="生成摘要中..."    // 当前状态描述
  stage="生成摘要"          // 当前阶段名称
  details={{                // 可选的详细信息
    downloaded: "750 MB",  // 已下载大小
    total: "1000 MB",      // 总大小
    speed: "2.3 MB/s",     // 下载速度
    remaining: "120秒"     // 剩余时间
  }}
/>
```

## 消息通信协议

### Popup → Background

```typescript
// 检查模型状态
{
  type: 'CHECK_MODEL_STATUS'
}

// 开始分析
{
  type: 'SUMMARIZE_PAGE'
}
```

### Background → Popup

```typescript
// 进度更新
{
  type: 'PROGRESS_UPDATE',
  progress: 0.45,           // 0-1
  message: '加载模型中... 45%',
  stage: '下载模型权重文件',
  details: {
    downloaded: '450 MB',
    total: '1000 MB',
    speed: '2.3 MB/s',
    remaining: '240秒'
  }
}
```

## 关键代码位置

| 文件 | 功能 |
|------|------|
| `src/App.tsx` | Popup 主组件，监听进度更新 |
| `src/components/LoadingView.tsx` | 进度展示组件 |
| `src/background.ts` | Service Worker，发送进度更新 |
| `src/lib/webllm-manager.ts` | WebLLM 管理器，生成进度信息 |

## 性能优化

1. **进度节流** - 避免过度频繁的 UI 更新
2. **异步处理** - 不阻塞主线程
3. **错误恢复** - 网络失败时自动重试

## 调试技巧

### 查看进度日志

打开 Chrome DevTools：

```
1. 右键点击扩展图标
2. 选择"检查弹出窗口"
3. 查看 Console 标签
```

### 查看后台日志

```
1. 访问 chrome://extensions/
2. 找到扩展，点击"Service Worker"
3. 查看后台日志
```

日志格式：
```
[WebLLM] 开始初始化模型...
[WebLLM] 进度: 5.0% | 阶段: 初始化 WebGPU 环境
[WebLLM] 进度: 45.0% | 阶段: 下载模型权重文件 {downloaded: "450 MB", ...}
[WebLLM] 模型初始化完成
```

## 常见问题

### Q: 进度卡在某百分比不动

A: 这可能是由于：
- 网络下载缓慢
- GPU 编译需要时间
- 等待几分钟后会继续

### Q: 显示"WebGPU 不可用"

A: 确保：
- Chrome 版本 ≥ 113
- WebGPU 已启用（chrome://flags/）
- 硬件支持 WebGPU

### Q: 下载速度显示 0

A: 正常现象，可能在等待服务器响应或编译中
