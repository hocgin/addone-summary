# 安装引导流程说明

## 功能概述

扩展安装后会自动显示引导页面，帮助用户完成初始设置和模型下载。

## 组件说明

### 1. 欢迎页面 (welcome.html)

**触发时机：** 扩展首次安装后

**功能：**
- 展示扩展核心功能
- 提供快速开始指南
- 自动引导用户打开扩展设置

**特点：**
- 精美的 UI 设计
- 渐变背景效果
- 响应式布局

### 2. OnboardingView 组件

**触发时机：** 用户首次点击扩展图标

**4 个步骤：**

#### 步骤 1: 欢迎页
- 功能介绍卡片
- 特性说明
- "开始设置"按钮

#### 步骤 2: WebGPU 检查
- 自动检测 WebGPU 支持
- 显示检测结果
- 提供问题解决方法

#### 步骤 3: 模型下载
- 实时进度显示
- 下载信息（大小、速度）
- 后台下载选项

#### 步骤 4: 完成
- 使用指南
- "开始使用"按钮

### 3. App.tsx 集成

**状态管理：**
```typescript
type ViewMode = 'onboarding' | 'main'
```

**流程控制：**
1. 检查 `chrome.storage.local` 中的 `onboardingCompleted` 标志
2. 如果未完成，显示 OnboardingView
3. 如果已完成，显示正常的主界面

## 消息协议

### 新增消息类型

| 消息类型 | 方向 | 说明 |
|---------|------|------|
| CHECK_WEBGPU | Popup → Background | 检查 WebGPU 支持 |
| PRELOAD_MODEL | Popup → Background | 预加载模型 |
| MODEL_DOWNLOAD_PROGRESS | Background → Popup | 模型下载进度 |

### CHECK_WEBGPU

**请求：**
```typescript
{
  type: 'CHECK_WEBGPU'
}
```

**响应：**
```typescript
{
  supported: boolean
  message?: string
}
```

### PRELOAD_MODEL

**请求：**
```typescript
{
  type: 'PRELOAD_MODEL'
}
```

**响应：**
```typescript
{
  success: boolean
  error?: string
}
```

### MODEL_DOWNLOAD_PROGRESS

**进度更新：**
```typescript
{
  type: 'MODEL_DOWNLOAD_PROGRESS'
  progress: number  // 0-1
  stage?: string
}
```

## 测试步骤

### 1. 测试安装流程

```bash
# 1. 卸载旧版本（如果已安装）
# chrome://extensions/ → 移除扩展

# 2. 重新构建
pnpm build

# 3. 重新加载扩展
# chrome://extensions/ → 加载 dist/

# 4. 观察是否自动打开欢迎页面
```

### 2. 测试引导流程

1. **欢迎页面**
   - 检查页面样式
   - 点击"打开扩展设置"按钮

2. **Onboarding 步骤 1**
   - 查看功能介绍
   - 点击"开始设置"

3. **Onboarding 步骤 2**
   - 观察 WebGPU 检查结果
   - 根据结果选择操作

4. **Onboarding 步骤 3**
   - 观察下载进度
   - 测试"后台下载"

5. **Onboarding 步骤 4**
   - 查看使用指南
   - 点击"开始使用"

### 3. 测试后续使用

1. 关闭 popup
2. 重新点击扩展图标
3. 应该直接显示欢迎界面（WelcomeView）
4. 不再显示引导流程

### 4. 重置引导状态

如需重新测试引导流程，在 Console 中执行：

```javascript
chrome.storage.local.remove('onboardingCompleted')
```

## 常见问题

### Q: 安装后没有打开欢迎页面？

**A:** 检查 background Service Worker 是否正常运行：
1. 访问 `chrome://extensions/`
2. 找到扩展，查看是否有错误
3. 手动打开 `chrome-extension://<extension-id>/src/welcome.html`

### Q: Onboarding 卡在检查 WebGPU？

**A:** 确保 Chrome 版本 ≥ 113，并且 WebGPU 已启用

### Q: 模型下载失败？

**A:** 检查网络连接，稍后可以在主界面重新下载

### Q: 如何重新触发引导？

**A:** 执行 `chrome.storage.local.remove('onboardingCompleted')`

## 设计亮点

1. **渐进式引导** - 分步骤引导，不会让用户感到压力
2. **视觉反馈** - 清晰的图标和动画效果
3. **错误处理** - 友好的错误提示和解决方法
4. **灵活性** - 支持跳过，稍后完成
5. **状态保存** - 使用 chrome.storage.local 记录进度
